# Проектная работа "Веб-ларёк"

[![Vite](https://img.shields.io/badge/Vite-4.x-646CFF?style=flat&logo=vite&logoColor=white)](https://vitejs.dev/)
[![TypeScript](https://img.shields.io/badge/TypeScript-5.x-007ACC?style=flat&logo=typescript&logoColor=white)](https://www.typescriptlang.org/)
[![SCSS](https://img.shields.io/badge/SCSS-6.x-CC6699?style=flat&logo=sass&logoColor=white)](https://sass-lang.com/)

«Веб-ларёк» — это современный интернет-магазин для веб-разработчиков, где пользователи могут просматривать каталог товаров, добавлять их в корзину, просматривать детали товаров в модальных окнах и оформлять заказы с выбором способа оплаты и доставки. Приложение построено на принципах MVP-архитектуры (Model-View-Presenter), обеспечивая четкое разделение логики данных, представления и бизнес-логики. Все взаимодействия реализуются через события, что делает код масштабируемым и легко поддерживаемым.

## Функционал приложения

### Основные возможности
- **Каталог товаров**: Отображение списка товаров в сетке с карточками (название, цена, категория, изображение). Клик по карточке открывает модальное окно с деталями.
- **Просмотр товара**: В модальном окне показана полная информация о товаре (описание, цена, категория). Кнопка "Купить" добавляет товар в корзину (если не добавлен ранее), после чего меняется на "Удалить из корзины". Если цена отсутствует (null), кнопка заблокирована с текстом "Недоступно". После действия модальное окно закрывается.
- **Корзина**: Иконка корзины в шапке с счётчиком товаров. Открывает модальное окно со списком товаров, ценами, общей суммой и кнопкой "Оформить". Если корзина пуста, отображается сообщение "Корзина пуста", а кнопка оформления деактивирована. Поддерживается удаление товаров из корзины.
- **Оформление заказа**:
  - **Шаг 1**: Выбор способа оплаты ("card" или "cash") и ввод адреса доставки. Валидация: способ оплаты выбран, адрес не пустой. При ошибках показываются сообщения.
  - **Шаг 2**: Ввод email и телефона. Валидация: поля не пустые.
  - После успешного оформления отправляется POST-запрос на сервер, отображается модальное окно успеха с суммой списания, корзина и данные покупателя очищаются.
- **Модальные окна**: Закрытие по клику вне окна, на крестик или Esc. Запрет скролла страницы во время открытия (класс `modal-open` на body).
- **Интеграция с сервером**: Получение каталога товаров (GET /product/), отправка заказа (POST /order/). Обработка ошибок и отображение в консоли.

### Технические особенности
- **Асинхронная загрузка данных**: Каталог загружается с сервера при запуске.
- **Валидация**: Минимальная проверка форм (непустые поля, выбранный способ оплаты) без дополнительных regex.
- **Изоляция слоёв**: Модели данных независимы от UI и API. Взаимодействие через события (EventEmitter).
- **Хранение состояний**: Модели хранят данные (каталог, корзина, покупатель), представления рендерят на основе состояний.

## Технологии

- **Язык**: TypeScript (для типизации и безопасности кода).
- **Сборщик**: Vite (быстрая разработка и сборка).
- **Стили**: SCSS (модульные стили с переменными и mixin).
- **API**: Fetch для запросов к серверу.
- **События**: Собственный EventEmitter для шины событий.
- **Компоненты**: Классы для UI (наследуют от базового Component), шаблоны клонируются из HTML.
- **Типы**: Все данные типизированы (IProduct, IBuyer, IOrderRequest и т.д.).
- **Дополнительно**: Нет внешних библиотек (чистый JS/TS), фокус на производительности и простоте.

## Установка и запуск

Для установки и запуска проекта выполните следующие команды:

```bash
npm install
npm run start
```

Или с использованием Yarn:

```bash
yarn
yarn start
```

Приложение откроется в браузере по адресу `http://localhost:3000` (или другой порт, указанный в консоли).

```bash
npm run build
```

Или:

```bash
yarn build
```

Собранные файлы появятся в папке `dist/`.

## Архитектура приложения

Приложение построено по парадигме MVP (Model-View-Presenter), где:
- **Model**: Хранит и манипулирует данными (независимо от UI и API).
- **View**: Отвечает за рендеринг и взаимодействие с пользователем.
- **Presenter**: Связывает Model и View через события, содержит бизнес-логику.

Взаимодействие осуществляется через события (EventEmitter). Модели генерируют события при изменениях данных, представления — при действиях пользователя. Presenter (в main.ts) обрабатывает события, обновляя состояния.

### Данные (Model)

Модели данных изолированы и отвечают за хранение/манипуляцию:
- **ProductCatalog**: Хранит массив товаров (IProduct[]). Методы: `setArrayProducts`, `getArrayProducts`, `getProduct(id)`.
- **Cart**: Управляет корзиной (добавление/удаление без дубликатов, расчёт суммы, очистка). Методы: `addItem`, `removeItem`, `getTotalPrice`, `getCartItems`, `isItemInCart`, `clearCart`.
- **Buyer**: Хранит данные покупателя (payment, email, phone, address). Методы: `setBuyerData`, `getDetails`, `clearBuyerData`, `validateField`, `validate`.

Данные инициализируются пустыми, заполняются методами. Нет зависимостей от UI/API.

### Слой коммуникации

- **ApiCommunication**: Класс для взаимодействия с сервером (композиция с Api). Методы:
  - `getProducts()`: GET /product/, возвращает IProductsResponse { items: IProduct[] }.
  - `createOrder(data: IOrderRequest)`: POST /order/, отправляет данные заказа.

Использует базовый класс Api для fetch-запросов с обработкой ошибок.

### Представления (View)

UI-компоненты наследуют от Component или BaseCardView/BaseFormView:
- **GridCard**: Карточка товара в каталоге (emit 'product/open').
- **ProductQuicklook**: Модалка детали товара (кнопка "Купить"/"Удалить", emit 'product/add' или 'remove', закрытие после действия).
- **CartPanel**: Панель корзины (список, total, удаление, emit 'basket/remove', 'basket/checkout').
- **CheckoutDetailsStep / CheckoutContactsStep**: Формы шагов оформления (валидация, emit 'order/step1/next', 'order/step2/pay').
- **OverlayDialog**: Универсальная модалка (открытие/закрытие, no-scroll).
- **CartToggleButton**: Кнопка корзины в шапке (счётчик, emit 'basket/open').

Компоненты рендерят на основе состояний, используют cloneTemplate для шаблонов.

### Базовый код

#### Класс Component<T>
Базовый класс для всех UI-компонентов. Принимает контейнер (HTMLElement).

Методы:
- `setImage(element: HTMLImageElement, src: string, alt?: string)`: Устанавливает изображение.
- `render(data?: Partial<T>): HTMLElement`: Обновляет компонент данными и возвращает контейнер.

#### Класс Api
Базовый клиент для HTTP-запросов.

Конструктор: `baseUrl: string`, `options: RequestInit` (headers по умолчанию).

Методы:
- `get<T>(uri: string)`: GET-запрос.
- `post<T>(uri: string, data: object, method: ApiPostMethods = 'POST')`: POST/PUT/DELETE.
- `handleResponse<T>(response: Response)`: Обработка ответа (JSON или reject при ошибке).

#### Класс EventEmitter
Брокер событий (паттерн "Наблюдатель") для связи слоёв.

Поля: `_events: Map<EventName, Set<Subscriber>>`.

Методы:
- `on<T>(event: EventName, callback: (data: T) => void)`: Подписка.
- `off(event: EventName, callback: Subscriber)`: Отписка.
- `emit<T>(event: string, data?: T)`: Инициация события.
- `onAll(callback: (event: EmitterEvent) => void)`: Подписка на все события.
- `offAll()`: Очистка подписок.
- `trigger<T>(event: string, context?: Partial<T>)`: Возвращает функцию-триггер.

### Шина событий (eventsBus.ts)
Определяет типы событий AppEvent (product/open, add, remove; basket/open, remove, checkout; order/step1/next, step2/pay; modal/close).

Экспортирует экземпляр EventEmitter как `events`.

## Разработка и отладка

- **Тестирование моделей**: В main.ts можно использовать тестовые данные из data.ts для проверки (шаг 3 ТЗ).
- **API**: Убедитесь, что сервер на `API_URL` доступен. Используйте Postman для проверки эндпоинтов.
- **Ошибки**: Логи в консоль при неудачных запросах.
- **Расширение**: Легко добавить новые события или компоненты благодаря MVP.
